# Contributing

Thanks for your interest in improving MAGNUS Reservation System.

## How to contribute

- Open an issue for bugs, security concerns, or feature requests.
- Keep pull requests focused on a single change where possible.
- Update documentation when behavior or configuration changes.
- Avoid committing secrets, personal data, or local environment files.

## Before submitting

- Check that your changes match the existing plain PHP structure.
- Keep business logic out of templates.
- Prefer secure defaults and privacy-friendly behavior.
- If you change configuration, schema, or deployment behavior, update `README.md` and any relevant example files.

## Security

If you find a security issue, avoid posting sensitive exploit details in a public issue before maintainers have had a chance to review it.
